# Brand Guidelines

Blink brand identity and usage guidelines.

## Brand Overview

**Blink** is a modern logistics and dispatch platform built for carriers and shippers. Our brand is:

- **Professional** — We handle serious logistics
- **Approachable** — We make complex things simple
- **Real** — We show actual work, not software screens
- **Empowering** — We solve real problems for drivers and dispatchers

---

## Tone & Voice

### Tone

How we sound in different contexts:

| Context | Tone | Example |
|---------|------|---------|
| **Buttons/CTAs** | Direct, action-oriented | "Book Now", "Create Run", "Export" |
| **Errors** | Helpful, not blaming | "Phone number format invalid. Try: +1 (555) 000-0000" |
| **Success** | Celebratory | "Booking confirmed! Driver assigned." |
| **Onboarding** | Welcoming, supportive | "Let's get you started. First, tell us about your business." |
| **Help/Support** | Patient, knowledgeable | "Need help? Here's how to use labels..." |

### Voice

Our consistent personality:

✓ **First-person active voice**
- ✓ "Book your pickup in 30 seconds"
- ✗ "The booking system allows rapid pickup scheduling"

✓ **Direct and clear**
- ✓ "Your driver is 3 min away"
- ✗ "Estimated time of arrival: 3 minutes"

✓ **Conversational, not robotic**
- ✓ "Something went wrong. Try again or contact support."
- ✗ "System error code 500: Internal server error"

✓ **Supportive**
- Acknowledge pain points
- Offer solutions
- Be human

✗ **Jargon-heavy**
- Avoid industry jargon
- Explain logistics simply

## Color Usage

### Primary Blue (#77c2f7)

**When to use:**
- Focus states (keyboard, mouse)
- Checked/selected states
- Active links
- Secondary CTAs (outline buttons)
- Badge backgrounds (informational)

**When NOT to use:**
- Page backgrounds
- Large text areas
- Primary action buttons (use Orange)

### Primary Orange (#FAA61A)

**When to use:**
- Primary action buttons (Book, Register, Create)
- Critical CTAs that need attention
- Key success indicators

**When NOT to use:**
- Links (use Blue)
- Error states (use Red)
- Decorative elements

### Dark Navy (#1A2332)

**When to use:**
- Header/navigation background
- Text on light backgrounds (primary text)
- Dark mode (if applicable)

**When NOT to use:**
- Light backgrounds (use text colors instead)
- Interactive elements

### Error Red (#ED5958)

**When to use:**
- Error messages
- Invalid form fields
- Destructive actions (delete, cancel)
- Alert states

**When NOT to use:**
- Success states (use Green)
- Information (use Blue)

### Success Green (#9ACF30)

**When to use:**
- Success messages
- Completed states
- Checkmarks
- Positive confirmations

**When NOT to use:**
- Errors (use Red)
- Links (use Blue)

## Imagery

### Photography Style

Show the **real work**, not the software:

✓ **DO:**
- Real drivers using the mobile app (POD)
- Dispatchers at desks with maps
- Actual trucks and vehicles (diverse types)
- Yards and dealerships in context
- People of different backgrounds
- Natural lighting
- Action shots (movement, progress)

✗ **DON'T:**
- Generic stock photos (people in suits)
- Outdated vehicles or equipment
- Computer screens in focus
- Paper-based workflows
- Competitors' branding
- Over-staged/posed shots

### Illustration Style

When using custom illustrations:

- **Consistent line weight** (not too thin, not too heavy)
- **Warm, professional color palette** (use brand colors)
- **Inviting, non-threatening style** (approachable)
- **Show workflow/action** (not static)
- **Diverse representation** (different people, vehicles, regions)

**Custom:**
- Commission 2-3 unique illustrations for key workflows

## Writing Guidelines

### Buttons & CTAs

- **Action-oriented verbs:** Book, Create, Export, Send, Confirm
- **Clear and concise:** 1-3 words max
- **Never apologetic:** ✗ "Sorry, please try again" → ✓ "Try again"

### Form Labels

- **Specific:** ✓ "Phone Number" not ✗ "Phone"
- **Required indicator:** Asterisk (*) for required fields
- **Helpful hints below field:** "Format: +1 (555) 000-0000"

### Error Messages

- **Be specific:** ✗ "Error" → ✓ "Email address already in use"
- **Offer solution:** ✗ "Invalid" → ✓ "Use 8+ characters, including uppercase"
- **Be kind:** ✗ "Wrong!" → ✓ "Doesn't match password"

### Empty States

- **Acknowledge:** "No bookings yet"
- **Encourage action:** "Create your first booking"
- **Provide path:** "Click 'New Booking' to get started"

### Success Messages

- **Celebratory:** "Booking confirmed! 🎉"
- **Next step:** "Driver assigned. You'll receive updates via SMS."
- **Link to action:** "View booking details or create another"

## Component Principles

1. **Consistency** — Same component looks same everywhere
2. **Clarity** — Never guess what a button does
3. **Feedback** — Every action has visible feedback (hover, loading, success/error)
4. **Accessibility** — Works for everyone (keyboard, screen readers, colorblind)
5. **Efficiency** — Minimal clicks to complete tasks

## Do's and Don'ts

### DO
✓ Use active voice  
✓ Be specific and clear  
✓ Show real work (not software screens)  
✓ Use brand colors intentionally  
✓ Test with actual users  
✓ Maintain consistency across all surfaces  

### DON'T
✗ Use passive voice  
✗ Be vague ("There was an error")  
✗ Show generic stock photos  
✗ Overuse colors (save Orange for primaries)  
✗ Assume users know industry jargon  
✗ Break from established patterns
